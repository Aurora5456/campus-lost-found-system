# 校园失物招领系统

这是一个基于 Python Flask 的期末软件大作业项目，保存学生、管理员、帖子和私信数据。项目包含学生端和管理员端，支持注册登录、发帖、分类与筛选、多图上传、状态管理、智能匹配、站内私信、未读提醒、邮件通知，以及完整的管理员后台（数据看板、用户管理、全站公告、私信监管、批量操作、任意帖子状态管理）。

## 数据库说明

项目支持三种数据库，由环境变量 `DATABASE_URL` 决定（应用会自动建表并写入种子数据）：

- **MySQL**（默认，适合本地课程演示）：不设置 `DATABASE_URL` 时，使用 `config.py` 中的 MySQL 配置。
- **SQLite**（零配置，适合快速运行）：设置 `DATABASE_URL=sqlite:///instance/dev.db` 即可，无需安装 MySQL，首次启动自动建表并写入种子数据。
- **PostgreSQL**（线上持久化，Render 采用）：设置 `DATABASE_URL=postgresql://...`。`render.yaml` 已声明一个 free 计划的 PostgreSQL 数据库并自动注入连接串，数据**不会**随服务重启/重新部署而丢失（种子数据仅在首次为空时写入，幂等）。

## 技术栈

- Python Flask
- Flask-SQLAlchemy
- MySQL + PyMySQL（本地默认）/ SQLite（零配置、线上部署）

- Chart.js（管理员数据看板图表）

## 配置 MySQL

默认配置在 `config.py` 中：

```text
数据库主机：localhost
数据库端口：3306
数据库名称：lost_found_system
数据库用户：root
数据库密码：123456
```

如果你的 MySQL 用户名或密码不是 `root / 123456`，请修改 `config.py` 中的 `DB_USER`、`DB_PASSWORD`，或设置环境变量 `DB_USER`、`DB_PASSWORD`、`DB_HOST`、`DB_PORT`、`DB_NAME`。

## 邮件通知（可选）

系统支持在有人发来私信、或学生申请找回密码时发送邮件。需要配置以下环境变量后才会真正发送，未配置时会自动降级（私信只走站内未读提醒，找回密码会直接在页面给出重置链接），不会报错：

```text
SMTP_HOST       SMTP 服务器地址，例如 smtp.qq.com
SMTP_PORT       端口，默认 587
SMTP_USER       发信邮箱账号
SMTP_PASSWORD   邮箱授权码 / 密码
SMTP_USE_TLS    是否使用 TLS，默认 true
MAIL_FROM       发件人显示名，例如 "校园失物招领系统 <no-reply@xxx.com>"
SITE_BASE_URL   站点外链地址，用于邮件里拼接链接，默认 http://127.0.0.1:5000
```

> 短信通知需要接入第三方服务商（阿里云短信、Twilio 等），`notifications.py` 中预留了 `send_sms` 接口，按需实现即可。

## 创建数据库

先登录 MySQL，然后执行：

```sql
CREATE DATABASE lost_found_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

也可以执行项目中的 `database.sql` 创建数据库和表结构。

## 初始化数据库

推荐双击或运行：

```bash
init_db.bat
```

也可以手动运行：

```bash
python init_db.py
```

该脚本会自动创建 `lost_found_system` 数据库，清空并重建数据表，然后插入测试学生账号、管理员账号和示例帖子。

如果提示 MySQL 连接失败，请检查：

- MySQL 服务是否已启动
- 是否已经创建 `lost_found_system` 数据库
- `config.py` 中的账号、密码和端口是否正确

## 启动项目

推荐使用一键启动脚本，它会后台启动 Flask、写入日志、检测端口并打开浏览器：

```bash
start.bat
```

停止服务：

```bash
stop.bat
```

手动启动方式：

```bash
python app.py
```

浏览器访问：

```text
http://127.0.0.1:5000
```

## 主要功能完成情况

- 学生注册、登录、退出、个人设置
- 找回密码（邮箱重置链接，未配置邮件时页面直接给出链接）
- 学生查看失物招领帖子列表（分页展示）
- 按标题、物品名称、描述、地点模糊搜索
- 按类型（失物/招领）、分类、状态多维筛选
- 学生发布帖子，支持物品分类选择
- 多图上传，使用 Pillow 自动压缩并生成缩略图
- 帖子状态管理：寻找中/招领中 ↔ 已找回/已归还
- 物品智能匹配：根据分类、名称、地点关键词推荐可能匹配的失物/招领
- 学生只能编辑和删除自己的帖子
- 帖子详情页展示发布人、物品信息、图片相册和匹配推荐
- 帖子详情页联系发布人
- 学生私信列表与未读消息角标
- 收到私信时邮件通知（可配置 SMTP）
- 聊天详情页发送和查看消息
- 管理员登录后台
- 管理员查看、搜索、筛选、编辑、删除所有帖子
- 管理员数据看板：帖子总量、已解决率、近 14 天趋势、类型与分类图表（Chart.js）
- 管理员用户管理：学生列表（发帖数/状态/搜索）、封禁/解封、重置密码、删除账号（被封禁者无法登录）
- 管理员全站公告：发布/删除/上下线公告，学生端首页顶部横幅展示
- 管理员私信监管：按内容搜索、查看收发件人与关联帖子、删除不当私信
- 管理员可一键修改任意帖子状态（寻找中↔已解决）
- 管理员批量操作：列表多选后批量删除 / 标记已解决 / 重新打开
- 删除帖子时同步清理关联私信和图片文件
- 密码使用 Werkzeug 哈希保存
- 后端路由进行权限校验

## 答辩演示流程

1. 打开 `http://127.0.0.1:5000`，点击“注册”创建一个新学生账号（或直接用 `20230001 / 123456` 登录）。
2. 在首页查看示例帖子，演示卡片式列表、分页，以及按类型/分类/状态的筛选和关键词搜索。
3. 点击“发布帖子”，填写信息并选择分类，一次上传多张 `jpg/png/gif` 图片，系统会自动压缩并生成缩略图。
4. 进入帖子详情页，展示图片相册、物品信息、状态标签，以及“可能匹配的招领/失物线索”智能推荐。
5. 在自己的帖子详情页点击“标记为已找回/已归还”，演示帖子状态流转。
6. 进入“我的帖子”，编辑帖子（可删除已有图片、追加新图片），说明学生只能修改自己的帖子。
7. 在“设置”里填写邮箱并开启邮件通知（如已配置 SMTP）。
8. 退出登录，使用另一个学生账号 `20230002 / 123456` 登录，打开对方帖子点击“联系发布人”发送私信。
9. 切回第一个账号，演示导航栏“我的私信”上的未读角标和聊天记录。
10. 演示“忘记密码”：用学号 + 绑定邮箱申请重置链接并设置新密码。
11. 退出学生账号，使用 `admin / admin123` 登录后台。
12. 在管理员后台按类型/分类/状态筛选帖子，编辑（含状态）或删除帖子。
13. 打开“数据看板”，展示帖子总量、已解决率、近 14 天新增趋势、类型与分类图表。
14. 打开 MySQL 客户端，查看 `students`、`posts`、`post_images`、`messages`、`password_reset_tokens` 等表的数据记录。

## 部署（线上 / 任意 WSGI 平台）

生产环境使用 `wsgi.py` 作为入口（默认 SQLite，首次启动自动建表 + 种子数据），用 Gunicorn 运行：

```bash
gunicorn wsgi:app
```

仓库内 `render.yaml` 是 Render 的一键部署蓝图（free 计划）。推送到部署分支后 Render 会自动重新构建上线。

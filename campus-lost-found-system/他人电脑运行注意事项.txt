校园失物招领系统 - 他人电脑运行注意事项

一、运行前必须准备

1. 安装 Python
   建议 Python 3.10 或以上版本。
   安装时请勾选 Add Python to PATH。

2. 安装 MySQL
   本项目必须使用 MySQL，不使用 SQLite。
   MySQL 服务需要保持启动状态。

3. 解压项目
   请把压缩包解压到一个路径简单的位置，例如：
   D:\lost_found_system
   不建议放在带特殊权限的系统目录。


二、安装依赖

在项目目录打开 CMD 或 PowerShell，运行：

pip install -r requirements.txt

如果下载较慢，可以换国内源，例如：

pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple


三、配置 MySQL 账号密码

项目默认 MySQL 配置在 config.py：

主机：localhost
端口：3306
数据库名：lost_found_system
用户名：root
密码：123456

如果组员电脑上的 MySQL 密码不是 123456，请打开 config.py 修改：

DB_USER = "root"
DB_PASSWORD = "你的MySQL密码"

也可以修改 DB_HOST、DB_PORT、DB_NAME。


四、初始化数据库

方法一：双击 init_db.bat

方法二：在项目目录运行：

python init_db.py

说明：
init_db.py 会自动创建 lost_found_system 数据库，自动创建 students、admins、posts、messages 四张表，并插入测试账号和示例帖子。

注意：
每次运行 init_db.py 会重建数据表，已有帖子和私信会被清空，演示前慎用。


五、启动系统

推荐双击 start.bat。

start.bat 会：
1. 进入项目目录
2. 后台启动 Flask 服务
3. 写入 backend.log 和 backend.err.log
4. 检测 5000 端口是否启动
5. 自动打开浏览器页面

浏览器访问地址：

http://127.0.0.1:5000

如果浏览器没有自动打开，可以手动输入上面的地址。


六、停止系统

双击 stop.bat。

或在项目目录运行：

stop.bat

在 Navicat 中连接本机 MySQL：

端口：3306
用户名：root
密码：填写本机 MySQL 密码

连接后刷新左侧数据库列表，找到：

lost_found_system

展开 Tables，应能看到：

students
admins
posts
messages


九、常见问题

1. 网页打不开
   先确认是否已经运行 start.bat。
   再检查 5000 端口是否被占用。
   可先运行 stop.bat，再重新运行 start.bat。

2. 登录时报数据库错误
   通常是 MySQL 没启动、密码不对、或者没有初始化数据库。
   请检查 config.py，然后运行 init_db.bat。

3. pip 安装依赖失败
   请检查网络，或使用清华源：
   pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

4. 页面样式没有更新
   浏览器按 Ctrl + F5 强制刷新。

5. 图片上传失败
   只允许 jpg、jpeg、png、gif 格式。
   图片会保存到 static/uploads 文件夹。


十、演示建议

1. 先运行 init_db.bat 初始化干净数据。
2. 运行 start.bat 启动系统。
3. 使用学生账号登录，演示帖子列表、搜索、发布、编辑、删除。
4. 换另一个学生账号登录，通过帖子联系发布人，演示私信。
5. 使用管理员账号登录后台，演示搜索、编辑、删除任意帖子。
6. 打开 Navicat，展示 lost_found_system 数据库和四张表。
